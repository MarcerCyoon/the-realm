December 1, 2019
Commands Command Prompt Cannot Understand
Like any normal person does, I spent the first day of December mostly on my computer. Recently, I had my almost five-year-old MacBook Pro break down on me, so I've been borrowing a Lenovo Thinkpad until my laptop arrives. I opened Command Prompt to set up a few repos on the new computer when, to my surprise, the 'ls' command refused to work. A quick Google search yields the answer: 'ls' is not a recognized command in Command Prompt! It's only a Unix command. You have to use the equivalent 'dir' while working in Command Prompt. As someone who's used Terminal for their entire programming career, this was a pretty neat new thing to learn.
Computer scientists and their love for inconsistencies, am I right?
